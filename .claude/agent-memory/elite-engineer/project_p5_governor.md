Phase 5 (code side) of the facebook-automation production conversion: the SAFETY-FIRST PACING GOVERNOR (user's explicit anti-ban priority) + carry-overs. All working-tree only (HARD CONSTRAINT: no commit; user tests before commit).

**Governor design (`core/governor.js`):** factory `createGovernor(settings, deps)` → `{ canAct(account) }` returning a DECISION OBJECT `{allowed, reason, detail}`, NEVER throws (a throw would bubble to the loop supervisor and turn "we paced ourselves" into a backoff restart). Checked cheapest-first: pacing master switch → active-hours window → per-account cap → global cap. Three semantic traps encoded correctly: (1) `0 = UNLIMITED` special-cased BEFORE the count query (naive `count>=0` would block everything); (2) `start===end` active-hours → full-day (always active), not empty-window; (3) wrapping/overnight window when `end<=start`. Active-hours uses ACCOUNT-local hour via `Intl.DateTimeFormat` + `account.timezoneId` (falls back to server-local on bad tz, never throws). Deps `{countActionsToday, now}` injectable → unit-testable with no DB/clock.

**Enforcement point:** `worker/loop.js` `checkAndAct` — a `governed(label, fn, targetUrl)` helper wraps each write action: asks `governor.canAct`, if denied logs a `skipped` action_log row + returns WITHOUT calling the action body, if allowed runs the `withRetry`-wrapped action + logs `ok`. The fragile `fb/actions/*` selector code is UNCHANGED (zero signature churn) — gating + logging live at the orchestration layer. MONITOR (post detection) is a READ — NOT gated, NOT cap-counted. The monitor (`fb/monitor.js`) gates per-comment reply+DM via the same governor (passed through `ctx`, optional — absent ctx = legacy behavior). `actionTypeFor(label)` maps withRetry labels (LIKE/COMMENT/SHARE/SHARE-GROUPS/DM) → action_log.action_type vocabulary.

**Sargable + TZ-correct cap query (`db.countActionsToday(accountId?)`):** the key trick — local-day window expressed BACK in UTC so `created_at` (stored UTC via `datetime('now')`) stays BARE in the predicate (sargable) AND correct across the server's UTC offset. Bounds: `datetime('now','localtime','start of day','utc')` .. `+1 day`. EXPLAIN confirms: global path → `idx_action_log_created`; per-account → `idx_action_log_acct_day`. Nullable accountId: with id = per-account, null/omitted = global. Only `status='ok'` counts toward caps (skipped/failed excluded). The pre-existing `date(created_at)=date('now')` form was BOTH non-sargable (function-wrapped created_at = full scan) AND mis-bucketed near midnight (compared UTC created_at to UTC date while active-hours is local).

**Heartbeat last-writer-wins fix:** `worker_state` is a SINGLE row (id=1); N accounts writing per-cycle status to it means one account's 'error' is clobbered by another's 'running' (the 90s liveness ticker also writes 'running'). Fix is ADDITIVE not replacement: new `account_status` table (one row/account, ON DELETE CASCADE) written by the loop at cycle boundaries via `db.setAccountStatus(id,status,detail)`; `worker_state` heartbeat KEPT for the control-plane's global process-liveness contract (`server/routes/worker.js:62` reads worker_state.status/last_heartbeat for the "alive within 90s?" check). `last_cycle_at` bumps only on terminal (ok|error), not on transient 'running'. Read via `getAccountStatus`/`listAccountStatuses`.

**Retention:** `db.trimActionLog(days)` — parameterized (`datetime('now', ?)` binds `'-Ndays'`, no interpolation). CATASTROPHIC GUARD: days<=0/NaN coerced to 30 (else `created_at < now` deletes EVERYTHING). Wired in `index.js` `startRetention()` (boot trim + daily unref'd interval), stopped in gracefulShutdown. N from env `ACTION_LOG_RETENTION_DAYS` (default 30) — kept OFF the settings serializer (server/ owned by parallel ops engineer). `index.js` got a `require.main === module` guard so tests can import the retention primitives without auto-running main()/launching browsers.

**Carry-over fix:** `fb/actions/comment.js` null-deref — `getActivePostScope` returns null on standalone-page posts; old code WARNED then unconditionally `dialog.$(sel)` → TypeError on every standalone-page comment. Fixed with `scope = dialog || page` fallback (the SAME pattern like.js:72 / share.js:43 already had — symmetric-drift gap closed).
